Authentication

Write operations require a Bearer token. Pass it in the Authorization header:

-H "Authorization: Bearer YOUR_TOKEN"

Rate Limits

EndpointsLimitWindow
GET /health, GET /games, GET /codes30 requestsper 60s per IP
POST / DELETE (auth required)unlimited—

Exceeding the limit returns 429 with retry_after in seconds.

GET /codes?game={slug}

Returns verified and unverified codes for a game.

POST /codes

Manually add a code. Requires auth.

BodyTypeDescription
codestringThe redeem code
gamestringGame slug
curl -X POST https://game-codes.wisp.uno/codes \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"code": "MYCODE123", "game": "wuwa"}'
Response 
{"id": 42, "code": "MYCODE123"}
DELETE /codes/{id}

Remove a code by its ID. Requires auth.

curl -X DELETE https://game-codes.wisp.uno/codes/42 \
  -H "Authorization: Bearer YOUR_TOKEN"
Response 
{"ok": true}
POST /update-codes

Force an immediate scrape of all source sites. Requires auth.

curl -X POST https://game-codes.wisp.uno/update-codes \
  -H "Authorization: Bearer YOUR_TOKEN"
Response 
{"ok": true}
POST /check-codes

Re-verify codes against game redemption APIs. Requires auth.

curl -X POST https://game-codes.wisp.uno/check-codes \
  -H "Authorization: Bearer YOUR_TOKEN"
Response 
{"ok": true}

Scheduling

JobWhenWhat it does
update-codesEvery hourScrapes news sites for new codes, expires old ones
check-codesDaily 1:30 AM (Asia/Taipei)Re-verifies known codes against game redemption APIs

Vercel skips the built-in scheduler. Use POST /update-codes via external cron, or rely on Render/WispByte to scrape.
